# 単体テスト仕様書

## 1. 目的

本テストは、既存のAmazon Linux 2（以下、AL2）踏み台機をAmazon Linux 2023（以下、AL2023）へ置換したこと、およびECS管理用EC2インスタンスのAMIを最適化AMIへ変更したことに伴い、
**変更点が意図通り反映されていること**と、**変更点以外の構成が完全に一致していること**を検証することを目的とする。

---

## 2. 対象範囲

### 2.1 変更対象

- 踏み台機
  - OS: AL2 → AL2023
  - EIPの付け替え
- ECS管理用EC2
  - AMI: 既存AMI → ECS最適化AMI

### 2.2 非変更対象（比較対象）

以下の項目はAL2環境と完全一致であることを前提とし、差分があれば原因確認を行う。

- サブネット
- セキュリティグループ（SG）
- ルートテーブル
- IAMロール
- ネットワークACL
- ホスト名／タグ
- インスタンスタイプ
- ストレージ構成

---

## 3. テスト観点

本テストでは、以下の観点に基づき網羅的に検証を実施する。

- 構成一致性：変更対象以外の設定が既存環境（AL2）と完全一致していること
- 変更点妥当性：AMI置換およびEIP切替が意図通り反映されていること
- ネットワーク疎通：セキュリティグループ単位での通信がすべて正常であること
- ECS動作：ECSクラスタおよびサービスが正常に稼働すること
- データ整合性：rsyncにより移行されたファイルが完全一致していること
- ミドルウェア状態：Dockerが正常にインストール・起動・自動起動設定されていること
- 運用継続性：踏み台機として従来通り利用可能であること

各観点の詳細は以下に示す。

## 4. テスト実施手順（詳細）

### 4.1 事前準備

1. 既存AL2踏み台機および既存ECS用EC2の設定情報を取得（aws cli / console）
   - `describe-instances`, `describe-security-groups`, `describe-route-tables`, `describe-iam-instance-profile-associations`
   - Launch Template: `describe-launch-templates` / `describe-launch-template-versions`
2. 新AL2023踏み台機および新ECS用EC2の同一項目を取得
3. 取得結果を比較用シートに転記（差分管理）

---

### 4.2 構成一致性テスト（共通）手順

**方針：本テストは原則としてAWS Management Console上での目視比較により実施すること（CLIは補助的に使用可）**

各項目ごとに以下を実施：

1. AWS Consoleにログイン
2. 旧環境（AL2 / 既存EC2）と新環境（AL2023 / 新EC2）をそれぞれ別タブで表示
3. 対象リソース（EC2 / SG / Subnet / IAM Role 等）を同時に開く
4. 設定項目を上から順に目視で比較
   - ID（SG ID / Subnet ID / Role ARN 等）
   - 設定値（ポート / CIDR / フラグ 等）
5. 完全一致であることを確認
6. 不一致があった場合、以下を記録
   - 差分内容（Before / After）
   - 意図的変更か否か
   - 影響有無
   - 対応方針

合格条件：AMI以外の差分が存在しないこと

---

### 4.3 AMI確認手順

#### 4.3.1 踏み台機（AL2023）

1. AWS Console → EC2 → Instances → 対象インスタンス選択
2. 「AMI ID」を確認
3. 指定されたAL2023のAMI IDと一致することを確認
4. SSHログイン後、以下を実行
   - `cat /etc/os-release`
5. "Amazon Linux 2023" が表示されることを確認

期待結果：指定AMIかつOSがAL2023であること

---

#### 4.3.2 ECS用EC2（Launch Template基準）

1. AWS Console → EC2 → Launch Templates
2. 対象テンプレートを選択
3. 使用中のVersionを確認（Default/Latest）
4. 該当Versionの「AMI ID」を確認
5. 指定されたECS最適化AMIと一致することを確認

期待結果：Launch Template上のAMIが指定値と一致

---

### 4.4 セキュリティグループ通信テスト手順（ルール単位）

**方針：SGの各インバウンド／アウトバウンドルールごとに1ケース作成し、個別に疎通確認を行う**

#### ■ テストケーステンプレート

- ケースID：SG-IN/OUT-XXX
- 対象：踏み台機 / ECS EC2
- 方向：Inbound / Outbound
- プロトコル：TCP/UDP
- ポート：XXX
- 送信元/宛先：CIDR または SG

**手順**

1. 接続元となる端末またはインスタンスを準備
2. 指定プロトコル・ポートで通信実施
   - SSH：TeraTerm / ssh コマンド
   - TCP疎通：`nc -zv <host> <port>`
   - HTTP/HTTPS：`curl -v`
3. 接続結果を確認
4. 必要に応じてサーバ側ログ（/var/log/secure 等）を確認

**期待結果**

- 許可ルール：通信成功
- 拒否前提：通信失敗（タイムアウト等）

---

#### ■ 代表ケース（記述例）

##### Case SG-IN-001：外部→踏み台 SSH

- プロトコル：TCP
- ポート：22
- 送信元：許可IP

手順：

1. クライアント端末からTeraTermを起動
2. EIPを指定してSSH接続
3. 認証情報を入力

期待結果：正常にログインできること

---

##### Case SG-OUT-002：踏み台→RDS(PostgreSQL)

- プロトコル：TCP
- ポート：5432

手順：

1. 踏み台へSSHログイン
2. 以下コマンド実行
   - `nc -zv <RDSエンドポイント> 5432`
3. 接続結果を確認

期待結果：succeeded が表示されること

---

※上記テンプレートに従い、**全SGルールを網羅的にケース化すること（省略不可）**

---

### 4.5 ECS動作確認手順

1. EC2へSSH接続
2. ECSエージェント状態確認
   - `systemctl status ecs`
3. AWS Console → ECS → Cluster を確認
4. 対象インスタンスが登録されていることを確認
5. サービス/タスクを起動
6. CloudWatch Logs を確認

期待結果：すべて正常稼働

---

### 4.6 踏み台機運用テスト手順

1. SSH接続確認（TeraTerm）
2. 内部リソース接続（DB / Private EC2）
3. DNS確認
   - `nslookup` / `dig`
4. パッケージ管理確認
   - `dnf update --assumeno`

期待結果：従来通り操作可能

## 5. 画面キャプチャ取得（証跡）ルール

### 5.1 目的

テスト実施内容のトレーサビリティおよびレビュー対応のため、すべての重要確認項目について証跡（スクリーンショット）を取得・保存する。

---

### 5.2 取得対象

以下の項目は必ずキャプチャを取得すること：

- 構成一致性確認（旧 vs 新 の比較画面）
- AMI確認（Instance詳細 / Launch Template）
- セキュリティグループ各ルール画面
- 疎通確認結果（ターミナル結果含む）
- ECSクラスタ状態 / タスク実行結果

---

### 5.3 取得方法

1. 原則としてAWS Management Consoleの画面を使用
2. 旧環境・新環境を並列表示（ブラウザ2画面）
3. 同一項目が一画面内で比較できるように調整
4. フルスクリーンまたは対象設定が明確に判別できる範囲でキャプチャ取得
5. 必要に応じてハイライト（枠線）を付与

---

### 5.4 ファイル命名規則

以下の命名規則に従うこと：

[区分]*[リソース名]*[確認項目]\_[連番].png

例：

- EC2\_Bastion\_Subnet\_01.png
- SG\_Bastion\_Inbound22\_01.png
- ECS\_Cluster\_Status\_01.png

---

### 5.5 保存場所

- プロジェクト共有フォルダ（例：SharePoint / S3 / 社内NAS）に保存
- テストケースID単位でフォルダ分割すること

例：
/TEST\_EVIDENCE/
└─ SG-IN-001/
└─ SG-OUT-002/

---

### 5.6 注意事項

- 個人情報・認証情報（IP・鍵・パスワード等）が含まれる場合はマスキングを行う
- 取得漏れがないようチェックリストと突合すること
- レビュー時に即提示できる状態で整理すること
